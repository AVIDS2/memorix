## 5. SQLite Canonical Store

All runtime-canonical data in one SQLite DB (`~/.memorix/data/memorix.db`):

| Table | Store Class | Generation Counter |
|-------|-------------|-------------------|
| `observations` | SqliteBackend | `storage_generation` in meta |
| `mini_skills` | MiniSkillSqliteStore | `mini_skills_generation` in meta |
| `sessions` | SessionSqliteStore | (no separate generation counter) |
| `graph_entities` + `graph_relations` | GraphSqliteStore | (no separate generation counter) |
| `team_agents/messages/tasks/locks` | TeamStore | `last_seen_obs_generation` per agent (no separate meta counter) |
| `pipeline_traces` | initTraceTable() | pruned by pruneOldTraces() |

Shared DB handle: `getDatabase(dataDir)` in `sqlite-db.ts` -- WAL mode, `foreign_keys = ON`, single connection.

When SQLite unavailable: all stores degrade to read-only empty backends (`DegradedBackend`, `SessionGracefulDegrade`, `MiniSkillGracefulDegrade`). `getBackendName()` returns `'degraded'`, not `'json'`.

## 10. Known Limitations (non-blocking)

- **OpenCode permission model**: `PermissionRejectedError` when writing to external directories in orchestrated mode. Not a Memorix bug — OpenCode requires workspace approval.
- **Codex reviewer performance**: Codex takes 10m+ for simple review tasks. Not a Memorix bug — Codex inference speed.
- **install-uninstall.test.ts flake**: 2 tests occasionally fail under full-suite load but pass in isolation. Root cause: shared temp-dir contention under parallel vitest workers. Non-blocking — does not affect product behavior.
