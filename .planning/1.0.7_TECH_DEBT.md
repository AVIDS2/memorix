# 1.0.7 Technical Debt Ledger — Final Stabilization State

> Last updated: 2026-04-16 (v1.0.7 stabilization closeout)

## Architecture Rules (unchanged)

1. **SQLite is the only canonical store for new product capabilities.**
2. **JSON/JSONL: migration source, export/import artifact, backup, debug dump, or config only. Forbidden as runtime writable store.**
3. **Orama is a retrieval/index layer, not the source of truth.**
4. **Every retrieval surface must be freshness-safe (`withFreshIndex`).**
5. **Each phase must retire debt, not only add capability.**

---

## D1. Observation Persistence Dual-Backend Debt

- status: **retired**
- scope: `ObservationStore`, `obs-store.ts`, `sqlite-store.ts`
- what changed:
  - `JsonBackend` (writable JSON fallback) replaced by `DegradedBackend` (read-only empty store)
  - `createObservationStore()` tries SQLite first; if unavailable, returns `DegradedBackend` — not `JsonBackend`
  - `observations.json` is now **migration source only** — `SqliteBackend.migrateFromJsonIfNeeded()` imports on first init
  - No runtime code writes `observations.json`
  - `json-store.ts` **deleted** in 1.0.7 stabilization — zero runtime/test references
  - `getBackendName()` return type narrowed to `'sqlite' | 'degraded'` (removed `'json'`)
- residual: none

## SQLite Canonical Store Summary

All runtime-canonical data lives in one SQLite database per data directory (`~/.memorix/data/memorix.db`):

| Table | Store Class | Generation Counter |
|-------|-------------|-------------------|
| `observations` | `SqliteBackend` | `storage_generation` in `meta` |
| `mini_skills` | `MiniSkillSqliteStore` | `mini_skills_generation` in `meta` |
| `sessions` | `SessionSqliteStore` | (shared DB generation) |
| `graph_entities` + `graph_relations` | `GraphSqliteStore` | (shared DB generation) |
| `team_agents` + `team_messages` + `team_tasks` + `team_locks` | `TeamStore` | `last_seen_obs_generation` per agent (no separate meta counter) |
| `pipeline_traces` | `initTraceTable()` | pruned by `pruneOldTraces()` |

Shared DB handle: `getDatabase(dataDir)` in `sqlite-db.ts` — WAL mode, single connection.
