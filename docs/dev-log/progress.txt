## Completed In This Branch
- Bumped root and workspace package versions to 1.1.6.
- Added root `npm run update-models` and `@memorix/ai` `update-models` scripts for intentional model catalog maintenance.
- Changed `@memorix/ai` build to run TypeScript compilation only; normal build/prepublish no longer contacts live model catalog APIs.
- Added model and image catalog shrink guards. Live refresh now refuses unexpectedly small generated catalogs unless `MEMORIX_ALLOW_MODEL_CATALOG_SHRINK=1` is set intentionally.
- Updated generated catalog headers, release docs, changelog, and lockfiles.
- Added release regression tests for model catalog publish safety.
- Fixed `memorix doctor` observation counts so diagnostics match the current project scope instead of reporting global flat-store totals.
- Updated Claude Code project/plugin guidance so fresh sessions use `memorix_project_context` before ad-hoc file/dev-log reads, and use `memorix context` as the reliable headless fallback when MCP tools are still hidden behind async startup.

## Known UX Notes
- Claude Code print-mode reports MCP servers as `pending` during async startup. Natural UX smoke showed a generic continuation prompt may read `docs/dev-log/progress.txt` and git before using Memorix unless the guidance is explicit. Diagnostic smoke confirmed Claude Code can discover and call `mcp__memorix__memorix_project_context` after the registry finishes loading, but natural UX smoke with the current `opus4.8` proxy mapping was more reliable when guidance used the `memorix context` CLI fallback instead of asking the model to discover deferred MCP tools itself. Product fix belongs in setup-installed rules/skills, not user-local Claude config mutation.
- The default stdio MCP profile is intentionally compact (`micro`, 7 tools). Use `memorix serve --mode lite|team|full` only when the agent needs the larger surface.
