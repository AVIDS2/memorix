## Known UX Notes
- Claude Code print-mode can report MCP servers as `pending` during async startup because tools may be dynamically discovered. Tests show Claude Code can call both a probe MCP tool and `mcp__memorix__memorix_project_context`. The product guidance now follows official Claude Code behavior: prefer MCP/tool discovery first, and use `memorix context --task` only as the fallback when MCP is actually unavailable, disabled, or not discoverable. Product fixes belong in setup-installed rules/skills and plugin MCP config, not user-local Claude config mutation.
- The default stdio MCP profile is intentionally compact (`micro`, 7 tools). Use `memorix serve --mode lite|team|full` only when the agent needs the larger surface.
