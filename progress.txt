## Non-negotiable constraints

- No retired development artifact, local Docker run, mixed-model run, or
  seeded Track B diagnostic is confirmatory evidence.
- A worker never receives the public case tree, private transition, oracle,
  raw precursor events, another condition's state, or host artifact paths.
- A generic hidden-test mount is not a black-box verifier and must not unlock a
  result.
- Lack of KVM or a single-model route is a hard stop for confirmation, not a
  reason to relabel a local smoke as evidence.
- The private F-drive routed-Claude discovery receipt is a mixed-stack route
  diagnostic. It cannot be used as a single-model preflight, trial, baseline,
  or paper result.
- A worker configuration hash and a client-visible model label are insufficient
  to prove a confirmatory model route. Only the separate relay signature plus
  matching worker telemetry can satisfy that gate.
- A successful baseline runtime preflight proves only that a comparison
  baseline is runnable. It does not admit a case or support an effectiveness
  claim.
- A local socket transport smoke proves JSONL controller mechanics only. It
  never proves KVM isolation, subject provenance, or private-oracle grading.
- PowerShell reserves `$HOME`; operational smoke scripts must use a distinct
  variable such as `$sandboxHome` so temporary controller homes stay on the
  external artifact drive.
- The 72-hour diagnostic report is private maintainer material. Its tiny public
  fixtures, mixed client stack, and one-run cells are intentionally excluded
  from future paper tables and statistical analysis.
- A power-plan smoke receipt with hypothetical inputs is a tooling check only.
  It must never determine the confirmatory cohort; only an outcome-independent
  reviewer-frozen envelope can do that.
- A passing local test suite, socket receipt, or controller policy is not KVM
  isolation evidence. A confirmatory result still needs a remote runner-issued
  isolation measurement, a trusted relay-issued final request inventory, and
  the independent human/source gates.
- A runtime-attestation code path and its temporary-key tests are not a remote
  runtime-manager deployment or a KVM certificate. A confirmatory permit must
  receive a fresh third-party runtime statement with a key disjoint from worker
  and relay trust roots for the exact worker-result hash.
- A versioned runtime measurement policy/receipt is an admission contract, not
  a substitute for the private KVM evidence artifacts it commits. The raw
  artifacts must remain on the independent runtime manager and be available to
  the authorized audit process.
- A passing public-artifact manifest audit proves only that the selected static
  files match a safe whitelist. It does not admit a source, run an agent,
  validate a private oracle, or turn a design-only release into empirical
  evidence.
- The final paper must keep this revised novelty boundary. Any stronger
  comparative statement requires a full-paper audit of the cited work and must
  not be inferred from a title, abstract, or benchmark name alone.
- Typed runtime receipts and signed-hash checks still do not prove that a
  writable remote controller is trustworthy or that private KVM/egress evidence
  was independently inspected. Trusted signer files and the pinned verifier
  binary must live on an immutable controller deployment, and raw evidence must
  remain in a separately auditable private store before any confirmatory run.

## 2026-07-23 frozen public cohort result

The public-reproducible cohort completed from the frozen v1 registry and plan:
12 local fixtures, four conditions, and three fixed seeds for 144 rows. The
result validator accepted all 144 rows, found zero invalid infrastructure rows,
one exact provider-reported model, one full tool-policy hash, and stable case
and oracle hashes across conditions. The immutable public analysis receipt is
stored only in the external artifact root.

The preregistered primary contrast is canonical Memorix versus no memory. Its
case-clustered success estimate is 97.2% versus 94.4%, a +2.8 point difference
with a 95% bootstrap interval of [-8.3, 16.7] and descriptive sign-flip p=1.0.
This is not a stable efficacy claim. Average resource values favor Memorix
directionally (about 3,881 fewer input tokens, 0.00187 lower provider cost, and
1.97 fewer seconds per case cluster), but every reported resource interval
crosses zero. Mem0 and AgentMemory are retained as descriptive secondary
baselines, not post-hoc primary hypotheses.

All six task failures were valid tool-step-limit outcomes, not verifier or
infrastructure failures. The opposing case patterns matter: Memorix was better
on the public header-policy transfer, but exhausted the bounded tool budget once
on the Go slug transfer. The fixture cohort is therefore useful evidence for
the boundary "memory is conditional", while real-repository, cross-model, and
confirmatory tracks remain unfinished.

The public analysis CLI now fails closed on matrix completeness, registry and
model identity, uniform tool policy, and per-case definition/oracle drift. The
research manuscript has a rendered public-results section and passed visual
inspection. LaTeX builds use the existing Git-bundled Perl only in the build
process PATH; no global dependency was installed.

## 2026-07-23 native hook-session formation diagnostic

The research harness now has a distinct `native-session` formation surface for
portable Claude Code command-hook captures. It refuses a normalized transcript
or a generic text replay: a private hook JSONL converter removes transcript
paths, requires all declared paths to remain below one Git workspace, replaces
that workspace with a portable token, scans the result for public-safety
violations, and commits source and canonical capture hashes. Both conversion
and formation reject a dirty or mismatched workspace snapshot.

Formation rehydrates only the workspace token in a clean isolated clone, calls
the real `memorix hook --agent claude` command once per ordered event with an
isolated data directory, then requires a separate bounded Memorix search to
find a declared storage-probe marker. Trial results bind the native capture
hash and reject trace-replay evidence on this surface; only no-memory and
Memorix conditions are allowed, so no baseline is falsely treated as having an
equivalent native hook write path.

One controlled local Claude Code smoke created an actual `PostToolUse` event,
confirmed that the product stored a searchable observation, converted the
private event into a portable capture after the precursor was frozen, and
formed that capture successfully in a fresh clone. This is a client/harness
integration smoke only. It contains one trivial edit, has no comparison, and
is excluded from all efficacy, baseline, cross-model, and paper results. Raw
hook events and temporary client settings remain outside the repository.
