## 2026-07-22: Track C replay hardening and real-path diagnostics

- Rebuilt the Cobra Track C bundle from the private raw streams after detecting
  that Claude's terminal `result` duplicated the preceding assistant message.
  The capture parser now records that omission explicitly; it never silently
  de-duplicates ordinary repeated content.
- Calibrated the common development context ceiling from 180 to 512 lexical
  proxy tokens before any validation/test enrollment. `event-suffix-v1` now
  rejects a header-only replay instead of launching a blank `last-n` control.
- A real local Claude Track C `last-n` smoke on the B capture completed the
  caller-ownership repair with clean command audit. A real Memorix canonical
  smoke then completed the same repair after trace formation and one logical
  search/detail round. Both runs use a mixed local client stack and remain
  development diagnostics, not comparison evidence.
- The initial canonical smoke exposed two adapter defects: it used the invalid
  Memorix type `session-event` and accepted MCP `isError` replies as writes.
  The adapter now maps trace roles to supported types, rejects every failed MCP
  call, sanitizes detail paths before injection, and verifies direct retrieval
  through the installed Memorix 1.2.1 CLI.
- Explicit Memorix stores are synchronously searchable. Canonical formation now
  records asynchronous claim/knowledge/workflow work as deferred rather than
  polling unrelated maintenance; a real no-model diagnostic reduced formation
  from roughly 46 seconds to 3.25 seconds while retaining the relevant
  invariant in a path-sanitized 512-token retrieval view.
- The Cobra source check no longer requires the reference patch's `append`
  spelling. Hidden behavior tests plus the old-owner prohibition accept both
  `append` and `make`/`copy` implementations.
