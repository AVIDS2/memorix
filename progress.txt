# Memorix Agent-Memory Research Progress
# Updated: 2026-07-22

## Completed

- Implemented and tested the research harness foundations: canonical Track B
  retrieval, Track C trace replay, adapter receipts, bounded context, model
  route accounting, action ledgers, blinded annotation, source-ledger audit,
  and result-schema validation.
- Defined a worker/vault, sealed-patch, KVM-backed confirmatory architecture.
  The current Windows workstation and current VPS do not satisfy its KVM gate,
  so neither can issue a confirmatory result.
- Audited source candidates and recorded offline environment receipts on F:.
  They remain source leads, not benchmark cases.
- Ran an independent leakage review of the original development corpus. Every
  early task exposed answer material through its public surface. All eight were
  withdrawn, all local outcomes were excluded, and raw files were moved to an
  external maintainer quarantine.
- Reset the public case registry to zero entries. Empty is now a valid registry
  state and is validated by tests.
- Added the sealed-task boundary in code and documentation: private-oracle
  cases require a private transition commitment, explicit public allowlist,
  external transition patch, and external oracle assets. A local
  `development-authoring-v1` overlay can verify deterministic authoring gates
  but cannot run an agent trial.

## In Progress

## Next

2. Run preregistered matched no-memory screening on isolated workers.
3. Freeze validation/test cases and execute the full baseline and ablation
   matrix.
4. Run statistics, failure analysis, artifact audit, and manuscript build.

## Non-negotiable constraints

- No retired development artifact, local Docker run, mixed-model run, or
  seeded Track B diagnostic is confirmatory evidence.
- A worker never receives the public case tree, private transition, oracle,
  raw precursor events, another condition's state, or host artifact paths.
- A generic hidden-test mount is not a black-box verifier and must not unlock a
  result.
- Lack of KVM or a single-model route is a hard stop for confirmation, not a
  reason to relabel a local smoke as evidence.
