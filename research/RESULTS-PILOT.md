# Pilot Diagnostics Ledger

Status: all pre-admission diagnostics are excluded from effect analysis.

This file records only what the early work established about the harness:

- source materialization, bounded retrieval adapters, model-route accounting,
  trace canonicalization, and command-contamination checks can be exercised
  locally;
- local agent runs may reveal implementation defects in those mechanisms; and
- a local run with a mixed provider route, public oracle, or non-isolated
  workspace is not evidence about a memory method.

An independent review found that every early development exercise exposed
answer material through its public tree, task wording, transition, or precursor
record. All associated runs, including successful repairs and matched local
pairs, are therefore retired. They must not be pooled, shown as pilot outcome
numbers, used to choose a favored method, or cited in the paper.

Raw artifacts remain only in a maintainer quarantine outside the repository so
that harness regressions can be diagnosed. They are not a reproducibility
artifact and will not be released as benchmark data. The next recorded result
may appear here only after a fresh case passes the sealed-task, private-oracle,
single-model, independent trace-review, and worker/vault-isolation gates.

The short `exploratory-72h` pilot defined in `PROTOCOL.md` is deliberately not
recorded here. Its private diagnostic artifacts may be used to repair the
runner or reject an unsafe case, but its outcome counts cannot enter this
ledger, the confirmatory registry, a method-selection decision, or the paper.

## Native hook integration smoke

One isolated Claude Code client smoke was run solely to verify the native
formation plumbing: an actual `PostToolUse` event reached Memorix's command
hook, produced a searchable observation, and was later converted into a
portable capture that formed successfully in a fresh matching checkout. It is
not a task-outcome row: the workspace contained one trivial edit, there was no
control condition, no baseline, no independent review, and no private oracle.
It is retained only as a regression signal for the capture/formation adapter.
Its raw client event, settings, and output remain in the external quarantine
and are excluded from all results, figures, method selection, and paper claims.

## Invalid Historical Client Probes

The public historical `backoff` source has an offline environment receipt, but
it is still a screening lead rather than a benchmark case. Two direct-client
predecessor probes against that source were invalidated before any transfer
task: one used a defective temporary UTF-8 forwarder, and one reached a client
budget limit without a final handoff or capture. These are harness findings,
not agent outcomes. They cannot be paired, summarized, or used to estimate a
memory effect. The corrected forwarder has its own byte-preservation smoke,
which remains an integration check rather than an effectiveness result.
