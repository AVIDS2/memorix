# Pilot Diagnostics Ledger

Status: all pre-admission diagnostics are excluded from effect analysis.

This file records only what the early work established about the harness:

- source materialization, bounded retrieval adapters, model-route accounting,
  trace canonicalization, and command-contamination checks can be exercised
  locally;
- local agent runs may reveal implementation defects in those mechanisms; and
- a local run with a mixed provider route, public oracle, or non-isolated
  workspace is not evidence about a memory method.

An independent review found that every early development exercise exposed
answer material through its public tree, task wording, transition, or precursor
record. All associated runs, including successful repairs and matched local
pairs, are therefore retired. They must not be pooled, shown as pilot outcome
numbers, used to choose a favored method, or cited in the paper.

Raw artifacts remain only in a maintainer quarantine outside the repository so
that harness regressions can be diagnosed. They are not a reproducibility
artifact and will not be released as benchmark data. The next recorded result
may appear here only after a fresh case passes the sealed-task, private-oracle,
single-model, independent trace-review, and worker/vault-isolation gates.
