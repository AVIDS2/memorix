\section{Benchmark Design}

Each future case begins from a license-audited repository revision and contains
a precursor session, a controlled post-snapshot transition, and a dependent
transfer task. Public workers receive only the public case bundle and their
condition-specific context. Private transitions, controller-only oracles,
reference repairs, and raw precursor events remain outside the worker boundary.

Case admission has two layers. Automated checks bind source origin, immutable
commit, license bytes, offline environment receipt, public case definition, and
private commitments. A separate two-reviewer human receipt assesses whether the
new transition is independent from public solution history, whether the proposed
predecessor dependency is plausible, and whether current source could already be
sufficient. Approval permits private authoring; it does not itself make a case
confirmatory.

The first real-repository source leads span TypeScript, Python, and Go, but no
real-repository source is currently enrolled as a benchmark case. The separate
public cohort uses local fixtures and is explicitly reported as such. This
conservative state prevents an unsafe public issue, patch, test, or task from
becoming an answer key merely because the surrounding harness can execute it.
